</form> Tag ( form — "форма" )
веб-бетте пішін орнатады.

Пішін пайдаланушы Мен сервер арасында деректер алмасуға арналған. Формалардың көлемі деректерді серверге жіберумен шектелмейді, клиенттік сценарийлерді қолдана отырып, форманың кез-келген элементіне қол жеткізуге, оны өзгертуге және өз қалауыңыз бойынша қолдануға болады.

Құжатта нысандардың кез-келген саны болуы мүмкін, бірақ бір уақытта серверге тек бір форма жіберілуі мүмкін. Осы себепті бұл формалар бір-бірінен тәуелсіз болуы керек.

Пішінді серверге жіберу үшін Submit батырмасы қолданылады, егер сіз пішін ішіндегі Enter пернесін бассаңыз, оған қол жеткізуге болады. Егер Submit түймесі формада болмаса, Enter пернесі оны қолдануды еліктейді.

Форма серверге жіберілген кезде, деректерді басқару </form> элементінің action төлсипатымен Берілген бағдарламаға жіберіледі. Алдын ала шолғыш ақпаратты "атау=мән" жұбы ретінде дайындайды, мұнда атау <input> элементінің name төлсипатымен анықталады және мәнді пайдаланушы енгізеді немесе әдепкі пішін өрісіне орнатады. Егер get әдісі Деректерді жіберу үшін пайдаланылса, онда мекен-жай жолы келесі көріністі қабылдай алады.

Параметрлер серверлік бағдарламаның мекен-жайынан кейін көрсетілген сұрақ белгісінен кейін тізімделеді және амперсанд (&) символымен өзара бөлінеді. Орыс әріптері он алтылық көрініске айналады (%HH түрінде, мұндағы HH — ASCII таңбасының мәні үшін он алтылық код), бос орын плюс (+) ауыстырылады.

</form> контейнерінің ішіне басқа элементтерді орналастыруға рұқсат етіледі, бұл ретте форманың өзі веб-бетте ешқандай түрде көрсетілмейді, тек оның ішіндегі элементтер ғана көрінеді.

</form> жұпты tag



Attributes

Бұл элемент үшін әмбебап атрибуттар қол жетімді.


accept-charset
Сервер пішін деректерін қабылдай және өңдей алатын кодтауды орнатады.

Syntax

<form accept-charset="мәні">
    ...
</form>

Мәні

Әдетті мәні
Кодтау атауы, Мысалы Windows-1251, UTF-8 және т. б. Бет үшін орнатылған кодтау.


action
Бұл нысандар серверге жіберілген кезде өңделетін өңдеушіні көрсетеді. Сервер бағдарламасы немесе серверлік сценарийлерді қамтитын HTML құжаты (мысалы, Parser) өңдеуші ретінде әрекет ете алады. Өңдегіш пішін деректерімен жұмыс істеу әрекеттерін орындағаннан кейін жаңа HTML құжатын қайтарады.

Егер әрекет төлсипаты болмаса, ағымдағы бет барлық пішін элементтерін әдепкі мәндеріне қайтару арқылы қайта жүктеледі.

Syntax

<form action="мәні">
    ...
</form>

Мәні

Әдетті мәні
Мән ретінде сервер файлының толық немесе салыстырмалы жолы қабылданады.
мекен-жайы 		Жоқ.


autocomplete
Пішін өрістерін автотолтыруды басқарады. Мәнді нақты пішін элементтеріндегі autocomplete атрибутымен жабуға болады.

Мәтіннің бірінші әріптерін енгізген кезде, қажетті мәндерді таңдауға болатын бұрын сақталған мәндердің тізімі көрсетіледі.

Syntax

<form autocomplete="мәні">
    ...
</form>

Мәні

Әдетті мәні
on
Нысанды автотолтыруды қамтиды.
off
Автотолтыруды өшіреді. Бұл мән әдетте браузерде маңызды деректерді (парольдер, банк карталарының нөмірлері), сондай-ақ сирек енгізілетін немесе бірегей деректерді (captcha) сақтауды болдырмау үшін қолданылады.
on


enctype
Серверге жіберілген кезде форма деректерін кодтау әдісін анықтайды. Әдетте enctype төлсипатының мәнін орнату қажет емес, деректер сервер жағында дұрыс түсініледі. Алайда, егер файлды жіберу үшін өріс қолданылса (input type="file"), enctype төлсипатын multipart/form-data ретінде анықтау керек.

Syntax

<form enctype="мәні">
    ...
</form>

Мәні

Әдетті мәні
application/x-www-form-urlencoded
Бос орындардың орнына +, орыс әріптері сияқты таңбалар олардың он алтылық мәндерімен кодталады (мысалы, Аня орнына %D0%90%D0%BD%D1%8F).
multipart/form-data
Деректер кодталмайды. Бұл мән файлдарды жіберу кезінде қолданылады.
text/plain
Бос орындар + белгісімен ауыстырылады, әріптер мен басқа таңбалар кодталмайды.
application/x-www-form-urlencoded

Мәні

Әдетті мәні
get
Бұл әдіс ең көп таралған әдістердің бірі болып табылады және қажетті ақпаратты алуға және мекен-жай жолында деректерді жіберуге арналған. "атау=мән" жұптары бұл жағдайда сұрақ белгісінен кейін мекен-жайға қосылады және амперсандпен (&символы) бөлінеді. get әдісін қолданудың ыңғайлылығы-барлық параметрлері бар мекенжайды бірнеше рет қолдануға болады, мысалы, оны шолғыш бетбелгілерінде сақтауға болады, сонымен қатар параметрлердің мәндерін тікелей мекенжай жолағында өзгертуге болады.
post
post әдісі браузер сұрауындағы деректерді серверге жібереді. Бұл get әдісіне қарағанда көбірек деректерді жіберуге мүмкіндік береді, өйткені post-та 4 Кб шегі жоқ. Деректердің үлкен көлемі форумдарда, пошта қызметтерінде, дерекқорды толтыруда, файлдарды жіберу кезінде және т. б. қолданылады.
get


name
Пішіннің ерекше атауын анықтайды. Әдетте, пішін атауы оның элементтеріне сценарийлер арқылы кіру үшін қолданылады.

Syntax

<form name="мәні">
    ...
</form>

Мәні

Әдетті мәні
Атау ретінде таңбалар жиынтығы, оның ішінде сандар мен әріптер қолданылады. JavaScript регистрге сезімтал, сондықтан сценарийлер арқылы формаға сілтеме жасаған кезде name атрибутындағыдай емлені орындаңыз. Жоқ.


novalidate
Пайдаланушы енгізген деректерді түзету үшін кірістірілген тексеруді болдырмайды. Мұндай тексеруді браузер пішінді серверге жіберген кезде автоматты түрде жүзеге асырады және <input type="email">, <input type="url"> өрістері үшін, сондай-ақ pattern немесе required төлсипаты болған кезде жүргізіледі.

Syntax

<form novalidate>
    ...
</form>

Мәні

Әдетті мәні
Жоқ. Әдепкі бойынша бұл мән өшірулі.


target
Пішін өңдегіші деректерді алғаннан кейін нәтижені HTML құжаты ретінде қайтарады. Сіз соңғы веб-бет жүктелетін жақтауды анықтай аласыз. Ол үшін target атрибуты қолданылады, оның мәні ретінде кадрдың атауы көрсетіледі. Егер target орнатылмаған болса, қайтарылған нәтиже ағымдағы қойындыда көрсетіледі.

Syntax

<form target="мәні">
    ...
</form>

_self
