сілтемелер жасауға арналған.

Ол үшін браузерге сілтеме не екенін айту керек, сонымен қатар сілтеме жасалатын құжаттың мекенжайын көрсету керек.

href атрибутының мәні ретінде өтпелі құжаттың мекен-жайы қолданылады. Сілтеме адресі абсолютті және салыстырмалы болуы мүмкін. Абсолютті мекен-жайлар сілтеме жазылған сайттың немесе Веб-беттің атауына қарамастан барлық жерде және барлық жерде жұмыс істейді. Салыстырмалы сілтемелер, олардың атауынан көрініп тұрғандай, ағымдағы құжатқа немесе сайттың түбіріне қатысты жасалған.

Бұл элемент үшін әмбебап атрибуттар қол жетімді.
download
download атрибуты болған кезде ( download — "жүктеу" ) браузер сілтеме бойынша өтпейді, бірақ сілтеме мекенжайында көрсетілген құжатты жүктеуді ұсынады.

Syntax

<a download="мәні">
    ...
</a>

Мәні

Әдетті мәні
Егер download төлсипаты мәнсіз жазылса, онда файл серверде жазылғандай бастапқы атымен жүктеледі және сақталады. Пайдаланушы дискісіне сақтау үшін ұсынылған файл атауын мән ретінде көрсетуге болады. Бұл жағдайда файл жүктеледі және download мәнінде көрсетілген атпен сақталады. Файл кеңейтімі бастапқы болып қалады. Әдепкі бойынша бұл мән өшірулі.
href
href атрибуты ( hypertext reference — "гипермәтіндік сілтеме" ) өту керек құжаттың мекенжайын көрсетеді. Кез-келген құжат сілтеме мекенжайы ретінде пайдаланылуы мүмкін болғандықтан, сілтеме бойынша өту нәтижесі соңғы файлға байланысты болады. Сонымен, мұрағаттар (zip немесе rar кеңейтімі бар файлдар) жергілікті дискіге сақталады. Әдепкі бойынша, жаңа құжат ағымдағы шолғыш терезесіне жүктеледі, бірақ бұл сипатты target атрибут арқылы өзгертуге болады.

Syntax

<a href="адрес">
    ...
</a>

Әдетті мәні

Мәні

Әдетті мәні
no-referrer
Referer тақырыбын жібермейді.
no-referrer-when-downgrade
Referer TLS тақырыбын (HTTPS) жоқ ресурсқа жібермейді. Бұл стандартты шолғыш әрекеті, егер басқа саясат көрсетілмесе.
origin
мекен-жай, хаттама, хост және порт сияқты ағымдағы бет туралы ақпаратты жібереді.
origin-when-cross-origin
басқа ресурстарға тек протоколды, хостты және портты жібереді, ал ресурс ішінде де жол жібереді.
unsafe-url
тек ресурс пен мекенжайды жібереді (бірақ парольдер немесе лақап аттар емес). Бұл мән қауіпсіз емес, өйткені ресурс пен мекен-жай TLS қорғалған ресурстарынан қауіпті ресурстарға ағып кетуі мүмкін.
no-referrer-when-downgrade
rel
rel атрибуты ( relation — "байланыс, қатынас" ) ағымдағы құжат пен href атрибутымен берілген сілтеме жүргізілетін құжат арасындағы қатынасты анықтайды. Көптеген браузерлер rel төлсипатын қолдамаса да, сайттарда Google және Яндекс іздеу жүйелеріне арналған rel="nofollow" кодын жиі кездестіруге болады. Осылайша белгіленген сілтемелер PageRank пен (ТИЦ)tic-ті жібермейді.

target
Әдепкі бойынша, сілтемені басқан кезде құжат ағымдағы шолғыш қойындысында ашылады. Бұл мінез-құлықты target атрибутының көмегімен өзгертуге болады ( target — "мақсат" ) </a> элементі. Мән ретінде </iframe> элементінің name атрибутымен немесе сақталған кілт сөзбен берілген кадр атауы қолданылады. Кадр атауын көрсеткен кезде сілтеме кадрда ашылады.
